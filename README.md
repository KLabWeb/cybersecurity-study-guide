# Application Security Engineering — Study Plan

A structured, self-directed plan for transitioning from full-stack web development into an entry-level **application security (AppSec) engineer** role, in roughly 7–9 months at about 21 hours per week.

The full plan lives in this repository as a PDF: **[Application_Security_Study_Plan.pdf](./Study_Plan_-_Final.pdf)**.

## Overview

This is the complete curriculum I'm following to make the switch into AppSec — the sequence of topics, hands-on labs, projects, resources, and certifications that take me from web-development foundations to interview-ready AppSec Eng. It is designed around producing public, reviewable work at every stage rather than passive study.

Two principles shape it:

- **Portfolio over checkboxes** - Each phase produces something concrete — vulnerability write-ups, a custom static-analysis rule, a full security assessment, a threat model, a secured API. As such, the work speaks for itself.
- **Credentials that carry weight** - The plan earns the **Burp Suite Certified Practitioner (BSCP)**, a hands-on exploitation exam, and **CompTIA Security+ (SY0-701)** for the baseline.
